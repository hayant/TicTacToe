# TicTacToe Game

A modern, full-stack TicTacToe game application featuring single-player AI opponents and real-time multiplayer gameplay. Built with ASP.NET Core and React, this application provides an engaging gaming experience with multiple game modes and difficulty levels. Additionally, this project serves as an AI development practicing platform, offering opportunities to experiment with and improve AI algorithms, game logic, and full-stack development patterns.

## Features

### Game Modes
- **Single Player**: Challenge AI opponents with 5 adjustable difficulty levels
- **Online Multiplayer**: Play against other players in real-time using SignalR

### Gameplay
- **20x20 Grid**: Larger playing field for extended strategic gameplay
- **5-in-a-Row Win Condition**: Connect five marks in a row (horizontal, vertical, or diagonal) to win
- **Real-time Updates**: Instant move synchronization in multiplayer mode
- **Game History**: All games are saved to the database for future reference

### AI Opponent
The AI opponent uses advanced algorithms including:
- Minimax with alpha-beta pruning for optimal move selection
- Iterative deepening governed by a per-move time budget
- Transposition table with bound flags (exact/lower/upper) and best-move ordering
- 64-bit Zobrist hashing for efficient board state identification
- Threat detection and blocking strategies
- Five difficulty levels (1-5) with adjustable search depth and candidate evaluation
- Thread-safe by design: each request runs on isolated search state, safe under concurrent SignalR calls

### User Features
- **Authentication**: Secure user registration and login system
- **User Management**: Track your game history and statistics
- **Online Lobby**: See connected players and send game requests

## Getting Started

### Prerequisites
- .NET 10.0 SDK
- Node.js and npm
- SQL Server (or SQL Server Express)

### Setup

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd TicTacToe
   ```

2. **Configure the database connection**
   - Update the connection string in `appsettings.json` or `appsettings.Development.json`
   - The application will automatically run migrations on startup

3. **Build and run the backend**
   ```bash
   cd TicTacToe
   dotnet restore
   dotnet build
   dotnet run
   ```

4. **Build the frontend** (in a separate terminal)
   ```bash
   cd TicTacToe/scripts
   npm install
   npm run build
   ```

5. **Access the application**
   - Login page: `http://localhost:5000/login`
   - Main application: `http://localhost:5000/app`
   - API documentation: `http://localhost:5000/swagger`

### Development

For frontend development with hot reload:
```bash
cd TicTacToe/scripts
npm start
```

## Testing

Unit tests for the AI engine live in `TicTacToe.Tests` (xUnit). They cover taking an immediate
win, blocking the opponent's immediate win, board immutability after a search (Apply/Undo and
Zobrist hash stability), deterministic results across repeated calls, and graceful behavior under a
tiny time budget.

Run them from the repository root:
```bash
dotnet test
```

## Continuous Integration

A GitHub Actions workflow (`.github/workflows/main_tictactoe.yml`) builds the frontend and backend,
runs the test suite, and publishes the app. It is also configured to deploy to Azure App Service.
