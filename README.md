# TicTacToe Game

A modern, full-stack TicTacToe game application featuring single-player AI opponents and real-time multiplayer gameplay. Built with ASP.NET Core and React, this application provides an engaging gaming experience with multiple game modes and difficulty levels.

## Features

### Game Modes
- **Single Player**: Challenge AI opponents with 5 adjustable difficulty levels
- **Online Multiplayer**: Play against other players in real-time using SignalR

### Gameplay
- **20x20 Grid**: Larger playing field for extended strategic gameplay
- **5-in-a-Row Win Condition**: Connect five marks in a row (horizontal, vertical, or diagonal) to win
- **Real-time Updates**: Instant move synchronization in multiplayer mode
- **Game History**: All games are saved to the database for future reference

### AI Opponent
The AI opponent uses advanced algorithms including:
- Minimax with alpha-beta pruning for optimal move selection
- Transposition tables for performance optimization
- Zobrist hashing for efficient board state evaluation
- Threat detection and blocking strategies
- Five difficulty levels (1-5) with adjustable search depth and candidate evaluation

### User Features
- **Authentication**: Secure user registration and login system
- **User Management**: Track your game history and statistics
- **Online Lobby**: See connected players and send game requests

## Technology Stack

### Backend
- **.NET 9.0**: Modern C# web framework
- **ASP.NET Core**: Web API and SignalR for real-time communication
- **Entity Framework Core**: Database ORM with SQL Server
- **SignalR**: Real-time bidirectional communication for multiplayer
- **Swagger/OpenAPI**: API documentation

### Frontend
- **React 19**: Modern UI library
- **TypeScript**: Type-safe JavaScript
- **Material-UI (MUI)**: Beautiful, responsive UI components
- **Webpack**: Module bundling and development server

### Database
- **SQL Server**: Relational database for user and game data

## Getting Started

### Prerequisites
- .NET 9.0 SDK
- Node.js and npm
- SQL Server (or SQL Server Express)

### Setup

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd TicTacToe
   ```

2. **Configure the database connection**
   - Update the connection string in `appsettings.json` or `appsettings.Development.json`
   - The application will automatically run migrations on startup

3. **Build and run the backend**
   ```bash
   cd TicTacToe
   dotnet restore
   dotnet build
   dotnet run
   ```

4. **Build the frontend** (in a separate terminal)
   ```bash
   cd TicTacToe/scripts
   npm install
   npm run build
   ```

5. **Access the application**
   - Login page: `http://localhost:5000/login`
   - Main application: `http://localhost:5000/app`
   - API documentation: `http://localhost:5000/swagger`

### Development

For frontend development with hot reload:
```bash
cd TicTacToe/scripts
npm start
```

## API Documentation

The API is documented using Swagger/OpenAPI. Once the application is running, visit `/swagger` to explore available endpoints and test the API.
